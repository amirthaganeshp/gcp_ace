1-Min Glimpse of Major GCP Services

  • STORAGE Need Global ACID + SQL? Choose Cloud Spanner. Need standard regional SQL? Choose Cloud SQL.
  • COMPUTE Standard vs Autopilot GKE: Autopilot handles nodes, sizes, and updates. Standard lets you configure nodes yourself. Choose Autopilot in scenario-based questions unless raw control is required.
  • NETWORKING Private Google Access lets GCE VMs with *only internal private IPs* securely access Google APIs (like GCS buckets) without going through the public internet.
  • SECURITY Never download service account keys in real-world scenarios. Attach service accounts directly to VM Metadata or use Workload Identity.
  • CLI bq mk creates BigQuery datasets/tables, while gsutil mb (or gcloud storage buckets create) creates GCS buckets.

Consensus Exam Tips & Strategy

  • Identify command prefixes: Remember `bq mk` creates BigQuery tables/datasets, `gsutil mb` or `gcloud storage` creates buckets, and `gcloud compute` manages VMs.
  • Always choose least privilege: For any IAM question, select the predefined role that offers the absolute narrowest permissions required. Never grant Owner/Editor roles if Predefined exists.
  • Autoscale and heal MIGs: Managed Instance Groups rely on *Instance Templates*. Startup scripts automatically install operations packages upon boot.
  • Understand firewall states: VPC Firewalls are stateful. Incoming Ingress rules automatically permit outgoing responses, while outgoing Egress rules automatically permit incoming requests.
  • Billing best practices: Set budgets and alerts at the billing level. Use exports to BigQuery for extensive querying and Looker Studio dashboards.

Reddit & Internet Consensus Weightages

Aggregated feedback from r/googlecloud candidates who passed the ACE certification reveals a heavy emphasis on identity controls, container management, and core networking:

1. IAM Security & Access Policy 25% - 30% Focus
Service accounts, Project folders, custom roles, and serviceAccountUser binds.
2. Container Orch & Compute Models (GKE, Run) 20% - 25% Focus
Autopilot vs Standard pools, HPA Pod configurations, and serverless Cloud Run APIs.
3. Storage & Relational Databases 20% - 25% Focus
GCS retention classes (Nearline/Coldline), Spanner synchronous writes, and SQL HA standby pools.
4. VPC Core Networks & Routing 15% - 20% Focus
Stateful firewall rule tags, subnets CIDR, Private Google Access, Peering, and VPN setups.
5. Operations monitoring & Custom billing 10% - 15% Focus
gcloud CLI script parameters, Log Router sinks, Ops Agent triggers, and budget notifications.
